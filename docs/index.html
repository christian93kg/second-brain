// 01 — the payoff

A well-run vault can answer four
questions at any moment

Most note systems accumulate. This one stays answerable. Ask it anything below and the structure already knows where to look.

q.01
What have I decided, and why?
q.02
What's still open, and what's blocking it?
q.03
Which facts are fresh, which are stale, which contradict each other?
q.04
What should I research next?

Claude does the bookkeeping — cross-referencing, staleness tracking, contradiction-hunting — that makes humans abandon wikis. You curate sources and ask good questions.

// 02 — the architecture

Information flows in one direction

Four layers. Each one cites the layer above it. Nothing points sideways or back up.

layer 0
Profile
who
→
layer 1
Claims
facts
→
layer 2
Analyses
compare
→
layer 3
Decisions
commit

Information flows strictly downward; each layer cites the one above it. That's what makes contradictions detectable and decisions reviewable.

You bring the content. The system files run the show.

// 04 — install · ~10 minutes

From template to live vault

Six steps. Most of them are one command.

~/my-brain — setup
1Use this template ▸ create your repo, then clone it:
$git clone https://github.com/<you>/my-brain.git ~/my-brain
2Install Obsidian, Claude Code, and the Obsidian CLI.
3Obsidian → Open folder as vault → pick ~/my-brain
4Start Claude inside the vault:
$cd ~/my-brain && claude
5Add the Obsidian skills:
›/plugin marketplace add kepano/obsidian-skills›/plugin install obsidian@obsidian-skills
6Tell Claude:
›"Walk me through BOOTSTRAP.md."
!
